모각코 2 Day   (2019.07.16)

모각코 2일차, 과연 오늘의 목표는??

오늘의 목표

8086 시스템 메모리와 CPU 레지스터 구조를 파악해보자.

           < Intel  8086  메모리 구조 >

시스템은 초기화 되면 커널을 메모리에 적재시키고 가용 메모리 영역을 확인한다.
32bit 시스템에서는 CPU가 한번에 처리 가능한 데이터가 32bit이다.
즉, 주소 할당 범위가 0 ~ 232 - 1 이다.

여기서 이제 알아야 하는 것은 하나의 프로그램 즉, 하나의 프로세스가 실행되기 위한 메모리 구조이다.
운영체제는 하나의 프로세스를 segment 단위로 묶어 가용 메모리 영역에 저장한다.

하나의 segment 는 위의 그림처럼 code segment, data segment, stack segment 로 이루어져 있다.
그리고 하나의 segment 는 최대 232 byte의 크기를 가질 수 있다.

  • Code segment  :
    메모리 상의 특정 위치에 시스템이 알아듣는 명령어(instruction)를 저장한다.
    기계어 코드로 컴파일러가 만든 코드이다.
    # 하지만 segment 는 자신의 physical address 를 모른다. 이 때, logical address 를 사용한다.
         segment selector 가 시작 위치 offset 을 찾고 그 위치부터 logical address 를 결정한다.
         즉,  physical address  =  offset  +  logical address 이다.

    ex)

    • 예를 들어 위에 메모리 상에선 segment 의 offset 은 0x80010000 (physical address) 이고, IS 1 은 0x00000100 (logical address) 이므로
    IS 1 의 physical address 는 0x80010000  +  0x00000100  =  0x80010100 이다.

  • Data segment  :
    프로그램 실행 시 사용되는 데이터(전역 변수)를 저장한다. 이 segment 는 다시 네 개로 나뉜다.
    현재 모듈의 data segment, 상위 레벨부터 오는 데이터 모듈, 동적 생성 데이터, 다른 프로그램과 공유하는 데이터로 나뉜다.

  • Stack segment  :
    LIFO(Last-In, First-Out) 방식으로 데이터를 명령어 PUSH 로 저장하고, POP 으로 사용한다.
    현재 수행되는 handler, task, program 이 저장하고, 지역 변수가 저장되는 영역이다.
    우리가 사용하는 buffer 가 이 영역에 저장된다. 또, multiple 스택 생성과 스택 간 switch 가 가능하다.
    필요 크기만큼 생성되고 프로세스 명령에 의해 데이터를 저장한다.
    SP(Stack Pointer) 레지스터가 스택 꼭대기를 가리킨다.


레지스터  :  CPU 내부 메모리

  • 범용 레지스터  :
    논리/수리 연산에 사용되는 피연산자, 주소 계산에 사용되는 피연산자, 메모리 포인터가 저장되는 레지스터이다.
    임의로 조작이 가능하며 4개의 32bit 변수처럼 사용된다.  또한, 때에 따라 다른 용도로 사용 가능하다.

    • EAX - 피연산자와 연산 결과 저장소
    EBX - DS segment  안에 데이터를 가리키는 포인터
    ECX - 문자열 처리나 루프를 위한 카운터
    EDX - I/O 포인터
    ESI - DS 레지스터가 가리키는 data segment  내의 어느 데이터를 가리키는 포인터, 문자열 처리에서 source 를 가리킴.
    EDI - ES 레지스터가 가리키는 data segment  내의 어느 데이터를 가리키는 포인터, 문자열 처리에서 destination 을 가리킴.
    ESP - SS 레지스터가 가리키는 stack segment 의 맨 꼭대기를 가리키는 포인터
    EBP - SS 레지스터가 가리키는 stack segment 의 한 데이터를 가리키는 포인터


  • 세그먼트 레지스터  :
    각각 code/data/stack  segment 를 가리키는 주소가 들어있는 레지스터이다.
    원하는 segment  안의 특정 데이터, 명령어를 정확하게 끄집어 낼 수 있다.

    • CS - code segment 를 가리키는 레지스터
    DS, ES, FS, GS - data segment 를 가리키는 레지스터
    SS - stack segment 를 가리키는 레지스터


  • 플래그 레지스터  :
    프로그램의 현재 상태나 조건 등을 검사하는 플래그가 있는 레지스터이다.
    상태, 컨트롤, 시스템 플래그의 집합이다. 시스템이 초기화 되면 이 레지스터는 0x00000002 값을 가진다.

    • 1, 3, 5, 15, 22~31 번 비트는 예약되어 있어서 조작할 수 없다.
    status flag, system flag 로 나뉜다. 종류는 많으나 필요할 때 찾아 보도록 하자.


  • 인스트럭션 포인터  :
    다음 수행해야 하는 명령의 메모리 상 주소(code segment 의 offset)가 있는 레지스터이다.
    EIP 레지스터는 소프트웨어에 의해 엑세스 할 수 없고 control-transfer instruction(JMP, Jcc, CALL, RET)이나
    interrupt, exception 에 의해 제어한다.
    EIP 레지스터를 읽는 방법은 CALL 을 수행한 뒤 procedure stack 으로 리턴하는 명령어의 주소를 읽는 것이다.
    procedure stack 의 return instruction pointer 의 값을 수정하고 return instruction 을 수행함으로 해서 EIP 레지스터 값을 간접적으로 지정 가능한다.



느낀점

시스템을 처음 입문하면서 많이 생소한 부분을 보았다. 프로그램(프로세스)이 실제로 메모리 상에서 어떻게 존재하는지 간단하지만 시각적으로 살펴보았다. 또, 레지스터 종류가 엄청 많은데 문서의 저자도 암기를 강요하지 않았고 필요할 때 다시 한 번 찾아 볼 수 있을 정도의 수준에서 공부하고자 한다.
앞으로 스택에 대한 이해도를 높이기 위해 열심히 노력하자.
